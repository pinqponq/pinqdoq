# Changelog

## api-endpoint-integration

- 2.1.0: Registration scripts now edit DI files in place instead of only printing instructions (a fork-from-MCP leftover that left files unwritten when no MCP consumed the output). `register_mapper.py`, `register_use_case.py`, and `register_di_modules.py` read the target module, insert the import + `singleOf(::X)` / module registration idempotently, and write — printing Target Path + Code only as a fallback when the file is missing (and exiting non-zero). `generate_use_case.py` now writes the use case file by default (only emits JSON with `--output-json`) and imports the inner domain type of generic returns like `List<Collection>`. `generate_mapper.py` accepts `domainField:responseField` (colon) as well as `=`. `generate_data_model.py` / `generate_domain_model.py` fail fast on a bare generic container type (e.g. shell-stripped `List`) with quoting guidance. `add_repository_impl.py` now edits the impl file in place (appending each method idempotently so repeated calls accumulate, injecting the mapper into the constructor once) and builds a `request: XRequest` body at any service arity while de-duplicating service params; it prints instructions only as a fallback when the impl file is missing. `path_utils.get_package_prefix` is now the single source of truth for package prefixes and prepends `config.base_package` consistently, so a non-empty `base_package` no longer produces mismatched packages between the layer scaffolds and the per-artifact/registration scripts (`base_package: ""` behavior is unchanged).
- 2.0.0: Forked into pinq-doq; decoupled from MCP and Figma. Every step now runs a standalone script under `.pinq-doq/scripts/` (`generate_data_layer.py`, `generate_domain_layer.py`, `generate_presentation_layer.py`, `generate_data_model.py`, `generate_domain_model.py`, `generate_mapper.py`, `generate_use_case.py`, `add_service_method.py`, `add_repository_method.py`, `add_repository_impl.py`, `register_mapper.py`, `register_use_case.py`, `register_di_modules.py`) invoked as `python .pinq-doq/scripts/<script>.py --project-root <path> ...` (layer scaffolds use `--config <path-to-your-config.json>`). Removed all Figma input/annotation/interaction steps and the get_design_context / figma-* tooling. Replaced the MCP discovery (`list-scripts`) with `ls .pinq-doq/scripts/` + per-script `--help`, and the `check-coding-standards` MCP review with a manual review against `common.md`. Clean Architecture flow, naming conventions, and the shared-module rules (same concrete artifact used by 2+ features) are preserved.
